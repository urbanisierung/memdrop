# Gallery Improvements Design

**Date:** 2026-04-13
**Branch:** memdrop-improvements1

---

## Overview

Five improvements to the memdrop app: gallery deletion from admin, infinite scroll for images, targeted performance fixes, upload button relocation, and dynamic per-event OG tags.

---

## 1. Delete Gallery (Admin)

### Backend

New endpoint: `DELETE /api/admin/events/{id}` (requires JWT).

Steps:
1. Query all image rows for the event from D1
2. Delete each image from R2: `events/{id}/orig/{imageId}` and `events/{id}/thumb/{imageId}.jpg`
3. Delete all image rows from D1 (`DELETE FROM images WHERE event_id = ?`)
4. Delete the event row from D1 (`DELETE FROM events WHERE id = ?`)
5. Return 204 No Content

### Frontend

- Add "Delete" button to each event row in `AdminPanel`
- On click: swap the button row to an inline confirmation — "Delete? [Confirm] [Cancel]" — no modal
- On confirm: call `deleteEvent(id)` action on the admin store, which calls `DELETE /api/admin/events/{id}`
- On success: remove the event from the `events` array in local Zustand state
- On error: show inline error text next to the buttons

### Admin Store Changes

Add `deleteEvent(id: string): Promise<void>` action that calls the new endpoint and removes the event from state.

## 2. Infinite Scroll / Lazy Loading

### Backend

Modify `GET /api/events/{eventId}/images` to accept pagination query params:
- `?limit=20&offset=0` (defaults: limit=20, offset=0)
- Response shape changes from `Image[]` to `{ images: Image[], total: number }`

### Frontend

**Event Store changes:**
- Add `total: number` to state (total image count from API)
- Add `loadingMore: boolean` to state
- Add `loadMore(eventId: string): Promise<void>` action — fetches next page using current `images.length` as offset, appends to state
- `load()` fetches first page only, sets `total`

**PhotoGrid changes:**
- Render a sentinel `<div>` after the last image
- Attach an `IntersectionObserver` to the sentinel
- When sentinel enters viewport and `images.length < total` and not `loadingMore`: call `loadMore(eventId)`
- Show a small spinner below the grid while `loadingMore` is true
- Observer is disconnected when `images.length >= total`

## 3. Performance Improvements

Targeted fixes only — no speculative work.

1. **Layout shift (CLS):** Add explicit `width` and `height` attributes to thumbnail `<img>` tags in `PhotoGrid`. Value: fixed grid cell dimensions already set in CSS (use `100%` / aspect-ratio approach).
2. **API cache headers:** Add `Cache-Control: public, max-age=60` to the images list response in `GET /api/events/{eventId}/images`. Images themselves already have `max-age=31536000, immutable`.
3. **Pagination (Section 2):** Eliminates the main performance problem — loading all images on mount.

---

## 4. Upload Button at Bottom

In `GalleryView`, move `<UploadZone>` to render after `<PhotoGrid>` instead of before it. No CSS changes required — existing layout already stacks vertically. This places the upload button at the bottom of the page, reachable with one hand on mobile.

## 5. Dynamic OG Tags

### Problem

Hash-based SPA routing means crawlers never see JS-rendered content. OG meta tags set by Preact are invisible to social platform scrapers.

### Hash Routing Constraint

The app uses hash routing (`/#/events/:id`). Hash fragments are never sent to the server — a crawler visiting `https://host/#/events/abc` only sends `GET /` to the server, making server-side OG injection impossible for hash URLs.

**Fix:** Switch gallery share links from hash URLs to path URLs. The "Copy link" button in `AdminPanel` will generate `https://host/events/{id}` (no hash). The SPA router is updated to read `window.location.pathname` in addition to `window.location.hash`.

### Solution: Bot-detection Pages Function

New file: `functions/events/[id].ts`

- Intercepts requests to `/events/:id` (path-based, generated by the "Copy link" button)
- Reads the `User-Agent` header
- If bot (Twitterbot, Slackbot, TelegramBot, facebookexternalhit, LinkedInBot, Discordbot, WhatsApp, etc.):
  1. Fetch event from D1 by `id`
  2. If event not found: pass through to SPA (index.html with base OG tags)
  3. If found: fetch `index.html` via `env.ASSETS.fetch()`, inject event-specific OG `<meta>` tags before `</head>`, return modified HTML
- If regular browser: serve `index.html` via `env.ASSETS.fetch()` (SPA handles routing via updated router)

### Router Update

`src/router.ts` currently reads only `window.location.hash`. Update to also check `window.location.pathname`:
- `/events/:id` path → same as `#/events/:id` hash → show GalleryView
- Existing hash routes continue to work unchanged (backwards compatible)

OG tags injected for bot requests:
```html
<meta property="og:title" content="{event.name}" />
<meta property="og:description" content="View photos from {event.name}" />
<meta property="og:image" content="https://{host}/api/events/{id}/og-image" />
<meta property="og:type" content="website" />
<meta property="og:url" content="https://{host}/events/{id}" />
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:title" content="{event.name}" />
<meta name="twitter:image" content="https://{host}/api/events/{id}/og-image" />
```

Base OG tags (always present in `index.html`):
```html
<meta property="og:site_name" content="memdrop" />
<meta property="og:title" content="memdrop" />
<meta property="og:description" content="Share photos from your events" />
<meta property="og:image" content="/favicon.svg" />
```

### OG Image Endpoint

New file: `functions/api/events/[id]/og-image.ts`

- `GET /api/events/{id}/og-image`
- Fetches event name from D1
- Returns dynamically generated SVG:
  - 1200×630px (standard OG image dimensions)
  - memdrop blue (`#4f8ef7`) background
  - Drop shape (from favicon, scaled up) on left
  - Event name in white, large font, centered
  - "memdrop" wordmark at bottom
  - Content-Type: `image/svg+xml`
  - Cache-Control: `public, max-age=3600`
- If event not found: return 404

### Why SVG

SVG OG images work on Discord, Telegram, Twitter/X, LinkedIn, Slack, iMessage. Facebook is the notable exception. Avoids `satori`/`resvg-wasm` dependencies (~500KB+ WASM). Acceptable tradeoff.
